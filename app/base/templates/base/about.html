About Issue Tracker

Issue Tracker is a Python web application running on Ubuntu. It uses the Django web framework to clone basic bug tracking features from GitHub and adds the ability to flag issues against folders, files, and individual lines of code.

The site is integrated with GitHub's public API. It utilizes the Python client, PyGitHub, and a custom client script to create, update, open, and close GitHub issues.

More details about the site can be found in the project README on GitHub.

Issue Tracker uses Django REST serializers to consume API data from GitHub. It can import a GitHub repository folder/file structure with all the repo's issues, including additional issue data added by Issue Tracker.

When adding additional issue data not tracked by GitHub, Issue Tracker "stamps" the Issue body with the data. Whenever Issue Tracker serializes a GitHub issue, it reads the stamp back into the appropriate foreign key relationships.

Issue Tracker comes with a REST API that displays its database in JSON format. View it here.

Issue Tracker runs on an AWS EC2 instance, using Apache to serve site files. It stores its data in a Postgres database housed on an AWS RDS instance. More details about the deployment process can be found here.
